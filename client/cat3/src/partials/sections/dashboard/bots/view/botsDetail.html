CATALYST_USER_DELETE

  • Used to delete an existing user from RL Catalyst Application. Users need to provide an existing user’s name and email ID to initiate the deletion. BOT will delete the user from the specified catalyst instance.
  • BOT Execution invoked either from “Ticketing System – ServiceNow” or from BOT Orchestrator.
  • If a “Ticket ID” is passed as parameter BOT executes as part of the Ticket workflow. If no “Ticket ID” is passed, BOT automatically creates a Ticket for “Catalyst User Deletion request” and stores the Ticket Context.
  • Parameters passed to BOT
    • Catalyst Instance URL (by Default Current Instance and Access Credentials assumed).
    • Access Credentials
    • User ID
    • Email of the existing user
    • BOT Type – Drop-down (UI BOT, Agent BOT options)
